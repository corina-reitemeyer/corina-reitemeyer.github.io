Sprint 3 - Javascript Fundamentals

An intro to JavaScript for non-technical folks.

5 April 2024

What is JavaScript's relationship to HTML and CSS?

Think about your nearest supermarket (because who hasn’t been to one?):

  • HTML is the building and the items inside (e.g. the shelves and products to be sold to customers).
  • CSS is the layout of the shelves inside the store, the paint, decorations, perhaps even the music playing in the background to make the experience less dull and boring.

So how does Javascript fit in? In this case it could be compared to the staff that operates the store (stocking shelfs, servicing you at the checkout, moving things in and out of the stock area in the back of the store etc.), just so you can walk in and buy things.

What is control flow and loops?

Control flow in Javascript is the order in how the computer runs code statements - usually it runs from the first line at the top all the way to the last line at the bottom, unless it encounters any statements that changes this such as loops, functions or conditionals. Imagine this like reading a book - you start reading from the top of the page all the way to the bottom.

Loops are also known as iteration statements, which will keep running until the condition becomes false or there is nothing to iterate through (and if there is, especially if it is *infinite*, your browser will most likely crash!). It is amazing for performing repetitive tasks efficiently without having to manually repeat or hardcode the same statement over and over again. To get back to the book example, imagine the author had copy and pasted the same sentence over and over and over again before continuing to the next paragraph. If you had no specific idea how many times this repetition would occur (and without warning), you would probably throw your book out the window at some point. But if you knew that this sentence would come up, e.g. 4 times, then you’d have some comfort knowing that something far more interesting would come up and continue the story.

What is the DOM and how to interact with it?

The Document Object Model (DOM) is an interface, which is a representation of the HTML & CSS of the page. The browser that you use to view a website represents the page as objects so that script-languages can interact with the document.

One of the best explanations I’ve seen on the internet so far was by FreeCodecamp.com, where they compared it to a TV remote. In order to fully use a television, whether its changing the program or modifying the volume, you’d need some sort of controller to do so.

Similar, because JavaScript by itself only does calculations and reads basic strings (aka the actions that would modification and change), it needs to access the content of the HTML page (aka the TV). This bridge happens via the DOM (aka the remote).

What is the difference between accessing data from arrays and objects?

Objects and arrays are pretty similar - they are considered a special data type that stores a collection of data rather than just a single value.

The difference becomes clear in the purpose of why you would want to use either one.

Objects are used to represent something in your code (e.g. a product, which then contains information about that product). Objects usually contain a set of characteristics, also called properties (consisting of a key and a value).

Arrays on the other hand can be thought of as “ordered collections”, a list of multiple items which are often related to one another, rather than a single variable.

The special abilities arrays have is that they can be accessed by their numerical position in the list. One must remember though that arrays are counted from 0 onwards, unlike us humans who start counting from 1.

What are functions and why they are helpful?

Functions are a set of statements that perform a task or calculates a value - however this tends to happen once and not in an multiple-iterative fashion (if that is what you want to do though, this is where loops come in handy!). To qualify as a function, it needs to have some type of input and then return an output.

The reason why they tend to be used often, is because they eliminate code complexity and enable reusability, meaning the code is concise and is used several times in the JavaScript program where it is defined. This means if changes are to be made, the risks of introducing bugs is lower.
